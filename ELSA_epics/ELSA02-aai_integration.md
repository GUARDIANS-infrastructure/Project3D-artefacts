# ELSA Epic 02: AAI solution integration

## Related GUARDIANS Task

3D-4-6: Defined core use-cases, requirements, acceptance criteria, and stub integration tests for integration with AAI solution (GUARDIANS project 2B), available on a platform like GitHub.

## User Story

- As a data steward, I want to login to ELSA with my institutional credentials, so that I can register a dataset, create and manage release.
- As a researcher, I want to login to REMS with my institutional credentials, so that I can review available release and retreive eventual file manifest.

## Context

- Data steward is invited to ELSA by the sys admin.
- Researcher is invited to ELSA by the sys admin.

## Acceptance Criteria

- [ ] Data steward from an AAF member organisation can login to ELSA via AAF AAI solution.
- [ ] Data steward from an organisation not AAF member can login to ELSA via AAF AAI solution.
- [ ] Researcher from AAF member organisation can login to ELSA via AAF AAI solution.
- [ ] Researcher from an organisation not AAF member can login to ELSA via AAF AAI solution.
